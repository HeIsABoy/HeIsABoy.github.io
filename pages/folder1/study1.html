# 使用场景

Nosql:降低磁盘io次数，越低越好；去除数据间关系，越简单越好。
作为关系型数据库的补充。
作用：应对基于海量用户和海量数据前提下的数据处理问题
常见的nosql数据库：redis memcache Hbase MongoDB

商品基本信息 ：名称 价格 厂商 =>MySQL
商品附加信息 ：描述 详情 评论 =>Nosql的MongoDB
图片信息 ：分布式文件系统
搜索关键字 ： ES Lucene solr
热点信息 ：高频信息 Redis

# redis常见数据类型

  • 字符串类型 String
  • 列表类型 LiSt
  • 散列类型 hash
  • 集合类型 set
  • 有序集合类型 sorted_set

# （1）字符串类型数据基本操作

  • 添加/修改数据
    set key value || mset key value key value
  • 获取数据 get key || mget key key （一次获取多个数据）
  • 删除数据
    del key
  • 获取数据字符个数
    strlen key
  • 追加信息到原始信息后部（如果原始信息存在就追加，否则就重建）
    append key value
  • 设置数值数据增加指定范围的值

incr key
incrby key incrment
incrbyfloat key incrment

  • 设置数值数据减少指定范围的值

decr key
decrby key increment

  • 设置数据具有指定的生命周期

setex key seconds value (setex tel 10 1) psetex key milliseconds value

  • 起名规范

第一种：
set user🆔00189:fans 19230385 表名：主键名：主键值：字段名
set user🆔00189:blogs 891
第二种:
set user🆔00189 {id:00789.blogs:789,.fans:19230385}

# （2）hash类型数据的基本操作

  • 添加/修改数据
    hset key field value || hmset key field1 value1 field2 value2（一次添加多个数据）
  • 获取数据

hget key feild || hmget key filed1 field2（一次获取多个数据）
hgetall key

  • 删除数据
    hdel key feild [filed2]
  • 获取哈希表中字段的数量
    hlen key
  • 获取哈希表中是否存在指定的字段
    hexists key feild
  • 获取哈希表中所有的字段名或字段值

hkeys key
hvais key

  • 设置指定字段的数值数据增加指定范围的值

hincrby key field increment
hincrbyfloat key field increment

# （3）list类型数据的基本操作 右进左查 朋友圈点赞场景

  • 添加/修改数据

lpush key value1 [value2]（一次添加多个数据）
rpush key value1 [value2]（一次添加多个数据）

  • 获取数据

lrange key start stop（start stop 是开始结束的数字 lrange list1 0 2）
lindex key index
llen key

  • 获取并移除数据

lpop key
rpop key

  • 规定时间内获取并移除数据

blpop key1 [key2] timeout
brpop key1 [key2] timeout

  • 移除指定数据
    lrem key count value
